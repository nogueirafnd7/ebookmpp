BUILDER

A Arte da Construção de Pink - Criando Performances Complexas

O que é o Padrão Builder?

O Padrão Builder é um padrão de projeto criacional que permite construir objetos complexos passo a passo. O padrão permite produzir diferentes tipos e representações de um objeto usando o mesmo código de construção [1].

É como a Pink construindo suas performances complexas - cada show é uma combinação única de elementos que podem ser montados de diferentes maneiras para criar experiências distintas.

Capa Próximo

O que ele resolve?

O principal problema que o Padrão Builder resolve é a criação de objetos complexos com muitos parâmetros ou configurações opcionais.

Sem este padrão, você teria construtores com muitos parâmetros ou múltiplos construtores sobrecarregados - como tentar planejar uma performance da Pink com todos os elementos de uma só vez [2].

O diagrama de classes UML a seguir ilustra a estrutura fundamental do Padrão Builder:

@startuml
class Director {
    - builder: Builder
    + construct(): void
}

interface Builder {
    + buildPartA(): void
    + buildPartB(): void
    + buildPartC(): void
    + getResult(): Product
}

class ConcreteBuilder1 {
    - product: Product1
    + buildPartA(): void
    + buildPartB(): void
    + buildPartC(): void
    + getResult(): Product1
}

class ConcreteBuilder2 {
    - product: Product2
    + buildPartA(): void
    + buildPartB(): void
    + buildPartC(): void
    + getResult(): Product2
}

class Product1 {
    + parts: List
    + listParts(): void
}

class Product2 {
    + parts: List
    + listParts(): void
}

Director --> Builder
Builder <|-- ConcreteBuilder1
Builder <|-- ConcreteBuilder2
ConcreteBuilder1 --> Product1
ConcreteBuilder2 --> Product2
@enduml
  • Director: Orquestra a construção usando a interface Builder
  • Builder: Interface para construir partes do produto
  • ConcreteBuilder: Implementações específicas de construção
  • Product: Objeto complexo sendo construído

Exemplo Fictício (A Arte de Pink)

Imagine que Pink constrói suas performances complexas com diferentes combinações de elementos:

Os Elementos de uma Performance da Pink:

  • Elementos Vocais: Power ballad, Rock anthem, Pop dance
  • Elementos Visuais: Acrobacias aéreas, Coreografias, Pirotecnia
  • Elementos Musicais: Banda ao vivo, Backing tracks, Orquestra
  • Elementos Cênicos: Palco giratório, Telões, Efeitos especiais

O Builder permite combinar esses elementos de diferentes maneiras para criar performances únicas sem repetir código.

Sem o Padrão: O que Acontece?

Sem o Padrão Builder, criar diferentes tipos de performances seria caótico:


// PROBLEMA: Construtores complexos e inflexíveis!
class Performance {
    constructor(tipo, acrobacias, pirotecnia, orquestra, 
                telaoGigante, palcoGiratorio, coreografiaComplexa, 
                duracao, intensidade, tema, ...mais20Parametros) {
        // Configuração extremamente complexa
    }
}

// Tentando criar diferentes performances:
const performanceAcrobatica = new Performance(
    "acrobatica", true, false, false, true, true, true, 
    120, "alta", "liberdade", ...mais20Parametros
);

const performanceIntima = new Performance(
    "intima", false, false, true, false, false, false,
    90, "media", "amor", ...mais20Parametros
);

const performanceEspecial = new Performance(
    "especial", true, true, true, true, true, true,
    180, "altissima", "anniversario", ...mais20Parametros

// Resultado: Código ilegível, difícil de manter e propenso a erros!
// E se quisermos uma performance com apenas alguns elementos?

Construtores com muitos parâmetros são difíceis de usar e manter!

Com o Padrão Aplicado (Pseudo Linguagem)

Aplicando o Padrão Builder, criamos performances complexas passo a passo:


// Product - O objeto complexo sendo construído
class Performance {
    constructor() {
        this.elementos = {
            vocais: [],
            visuais: [],
            musicais: [],
            cenicos: []
        };
        this.duracao = 0;
        this.intensidade = "media";
        this.tema = "";
    }
    
    adicionarElemento(tipo, elemento) {
        this.elementos[tipo].push(elemento);
    }
    
    descrever() {
        return `Performance ${this.tema} (${this.duracao}min, ${this.intensidade})`;
    }
}

// Builder Interface
interface PerformanceBuilder {
    reset(): void
    definirTema(tema: string): void
    construirElementosVocais(): void
    construirElementosVisuais(): void
    construirElementosMusicais(): void
    construirElementosCenicos(): void
    definirDuracao(duracao: number): void
    definirIntensidade(intensidade: string): void
    getPerformance(): Performance
}

// Concrete Builders
class PerformanceAcrobaticaBuilder implements PerformanceBuilder {
    private performance: Performance;
    
    constructor() {
        this.reset();
    }
    
    reset() {
        this.performance = new Performance();
    }
    
    definirTema(tema: string) {
        this.performance.tema = tema;
    }
    
    construirElementosVocais() {
        this.performance.adicionarElemento("vocais", "Power Ballad");
        this.performance.adicionarElemento("vocais", "Rock Anthem");
    }
    
    construirElementosVisuais() {
        this.performance.adicionarElemento("visuais", "Acrobacias Aéreas");
        this.performance.adicionarElemento("visuais", "Coreografia Complexa");
    }
    
    construirElementosMusicais() {
        this.performance.adicionarElemento("musicais", "Banda Ao Vivo");
    }
    
    construirElementosCenicos() {
        this.performance.adicionarElemento("cenicos", "Cabos Aéreos");
        this.performance.adicionarElemento("cenicos", "Palco Giratório");
    }
    
    definirDuracao(duracao: number) {
        this.performance.duracao = duracao;
    }
    
    definirIntensidade(intensidade: string) {
        this.performance.intensidade = intensidade;
    }
    
    getPerformance(): Performance {
        const resultado = this.performance;
        this.reset();
        return resultado;
    }
}

class PerformanceIntimaBuilder implements PerformanceBuilder {
    private performance: Performance;
    
    constructor() {
        this.reset();
    }
    
    reset() {
        this.performance = new Performance();
    }
    
    definirTema(tema: string) {
        this.performance.tema = tema;
    }
    
    construirElementosVocais() {
        this.performance.adicionarElemento("vocais", "Balada Íntima");
        this.performance.adicionarElemento("vocais", "Acústico");
    }
    
    construirElementosVisuais() {
        this.performance.adicionarElemento("visuais", "Iluminação Suave");
    }
    
    construirElementosMusicais() {
        this.performance.adicionarElemento("musicais", "Piano");
        this.performance.adicionarElemento("musicais", "Violão");
    }
    
    construirElementosCenicos() {
        this.performance.adicionarElemento("cenicos", "Palco Pequeno");
        this.performance.adicionarElemento("cenicos", "Projeções Artísticas");
    }
    
    definirDuracao(duracao: number) {
        this.performance.duracao = duracao;
    }
    
    definirIntensidade(intensidade: string) {
        this.performance.intensidade = intensidade;
    }
    
    getPerformance(): Performance {
        const resultado = this.performance;
        this.reset();
        return resultado;
    }
}

// Director
class DiretorDeShows {
    private builder: PerformanceBuilder;
    
    setBuilder(builder: PerformanceBuilder) {
        this.builder = builder;
    }
    
    construirShowAcrobatico() {
        this.builder.definirTema("Liberdade e Força");
        this.builder.definirDuracao(120);
        this.builder.definirIntensidade("alta");
        this.builder.construirElementosVocais();
        this.builder.construirElementosVisuais();
        this.builder.construirElementosMusicais();
        this.builder.construirElementosCenicos();
    }
    
    construirShowIntimo() {
        this.builder.definirTema("Amor e Vulnerabilidade");
        this.builder.definirDuracao(90);
        this.builder.definirIntensidade("media");
        this.builder.construirElementosVocais();
        this.builder.construirElementosVisuais();
        this.builder.construirElementosMusicais();
        this.builder.construirElementosCenicos();
    }
}

// Uso do sistema
const diretor = new DiretorDeShows();

// Construindo performance acrobática
const builderAcrobatico = new PerformanceAcrobaticaBuilder();
diretor.setBuilder(builderAcrobatico);
diretor.construirShowAcrobatico();
const performanceAcrobatica = builderAcrobatico.getPerformance();

// Construindo performance íntima
const builderIntimo = new PerformanceIntimaBuilder();
diretor.setBuilder(builderIntimo);
diretor.construirShowIntimo();
const performanceIntima = builderIntimo.getPerformance();

console.log(performanceAcrobatica.descrever());
console.log(performanceIntima.descrever());

Exemplo Prático do Dia a Dia (Python)

Um exemplo prático do Padrão Builder em um sistema de construção de relatórios:


from abc import ABC, abstractmethod
from typing import List

# Product
class Report:
    def __init__(self):
        self.parts = {
            'header': '',
            'body': [],
            'footer': '',
            'formatting': {},
            'metadata': {}
        }
    
    def add_body_part(self, part: str):
        self.parts['body'].append(part)
    
    def set_header(self, header: str):
        self.parts['header'] = header
    
    def set_footer(self, footer: str):
        self.parts['footer'] = footer
    
    def set_formatting(self, key: str, value: str):
        self.parts['formatting'][key] = value
    
    def set_metadata(self, key: str, value: str):
        self.parts['metadata'][key] = value
    
    def display(self):
        print("=" * 50)
        print(f"HEADER: {self.parts['header']}")
        print("-" * 50)
        print("BODY:")
        for part in self.parts['body']:
            print(f"  • {part}")
        print("-" * 50)
        print(f"FOOTER: {self.parts['footer']}")
        print(f"FORMATTING: {self.parts['formatting']}")
        print(f"METADATA: {self.parts['metadata']}")
        print("=" * 50)

# Builder Interface
class ReportBuilder(ABC):
    @abstractmethod
    def reset(self):
        pass
    
    @abstractmethod
    def build_header(self):
        pass
    
    @abstractmethod
    def build_body(self):
        pass
    
    @abstractmethod
    def build_footer(self):
        pass
    
    @abstractmethod
    def build_formatting(self):
        pass
    
    @abstractmethod
    def build_metadata(self):
        pass
    
    @abstractmethod
    def get_report(self) -> Report:
        pass

# Concrete Builders
class SalesReportBuilder(ReportBuilder):
    def __init__(self):
        self.reset()
    
    def reset(self):
        self._report = Report()
    
    def build_header(self):
        self._report.set_header("RELATÓRIO DE VENDAS - TRIMESTRAL")
    
    def build_body(self):
        self._report.add_body_part("Vendas totais: R$ 1.200.000,00")
        self._report.add_body_part("Crescimento: 15% vs período anterior")
        self._report.add_body_part("Produto mais vendido: Smartphone X")
        self._report.add_body_part("Região com melhor desempenho: Sudeste")
    
    def build_footer(self):
        self._report.set_footer("Relatório gerado automaticamente pelo sistema")
    
    def build_formatting(self):
        self._report.set_formatting("font", "Arial")
        self._report.set_formatting("color_scheme", "corporate")
        self._report.set_formatting("charts", "enabled")
    
    def build_metadata(self):
        self._report.set_metadata("author", "Sistema de Vendas")
        self._report.set_metadata("version", "2.1")
        self._report.set_metadata("confidential", "true")
    
    def get_report(self) -> Report:
        report = self._report
        self.reset()
        return report

class MarketingReportBuilder(ReportBuilder):
    def __init__(self):
        self.reset()
    
    def reset(self):
        self._report = Report()
    
    def build_header(self):
        self._report.set_header("RELATÓRIO DE MARKETING - CAMPANHA VERÃO")
    
    def build_body(self):
        self._report.add_body_part("Alcance total: 2.5 milhões de pessoas")
        self._report.add_body_part("Taxa de engajamento: 4.8%")
        self._report.add_body_part("Custo por clique: R$ 0,45")
        self._report.add_body_part("ROI: 320%")
        self._report.add_body_part("Canal mais eficiente: Instagram")
    
    def build_footer(self):
        self._report.set_footer("Dados coletados até 31/03/2024")
    
    def build_formatting(self):
        self._report.set_formatting("font", "Verdana")
        self._report.set_formatting("color_scheme", "creative")
        self._report.set_formatting("charts", "interactive")
    
    def build_metadata(self):
        self._report.set_metadata("author", "Time de Marketing")
        self._report.set_metadata("version", "1.5")
        self._report.set_metadata("confidential", "false")
    
    def get_report(self) -> Report:
        report = self._report
        self.reset()

class FinancialReportBuilder(ReportBuilder):
    def __init__(self):
        self.reset()
    
    def reset(self):
        self._report = Report()
    
    def build_header(self):
        self._report.set_header("RELATÓRIO FINANCEIRO - BALANÇO ANUAL")
    
    def build_body(self):
        self._report.add_body_part("Receita líquida: R$ 8.500.000,00")
        self._report.add_body_part("Lucro operacional: R$ 1.200.000,00")
        self._report.add_body_part("Margem líquida: 14.1%")
        self._report.add_body_part("Despesas operacionais: R$ 2.300.000,00")
        self._report.add_body_part("Investimentos: R$ 850.000,00")
    
    def build_footer(self):
        self._report.set_footer("Auditado por Contabilidade ABC Ltda.")
    
    def build_formatting(self):
        self._report.set_formatting("font", "Times New Roman")
        self._report.set_formatting("color_scheme", "professional")
        self._report.set_formatting("charts", "detailed")
    
    def build_metadata(self):
        self._report.set_metadata("author", "Departamento Financeiro")
        self._report.set_metadata("version", "3.0")
        self._report.set_metadata("confidential", "true")
    
    def get_report(self) -> Report:
        report = self._report
        self.reset()
        return report

# Director
class ReportDirector:
    def __init__(self):
        self._builder = None
    
    @property
    def builder(self) -> ReportBuilder:
        return self._builder
    
    @builder.setter
    def builder(self, builder: ReportBuilder):
        self._builder = builder
    
    def build_minimal_report(self):
        self.builder.build_header()
        self.builder.build_body()
    
    def build_standard_report(self):
        self.builder.build_header()
        self.builder.build_body()
        self.builder.build_footer()
        self.builder.build_formatting()
    
    def build_complete_report(self):
        self.builder.build_header()
        self.builder.build_body()
        self.builder.build_footer()
        self.builder.build_formatting()
        self.builder.build_metadata()

# Client code
def main():
    director = ReportDirector()
    
    # Construindo relatório de vendas completo
    print("=== RELATÓRIO DE VENDAS COMPLETO ===")
    sales_builder = SalesReportBuilder()
    director.builder = sales_builder
    director.build_complete_report()
    sales_report = sales_builder.get_report()
    sales_report.display()
    
    # Construindo relatório de marketing padrão
    print("\n=== RELATÓRIO DE MARKETING PADRÃO ===")
    marketing_builder = MarketingReportBuilder()
    director.builder = marketing_builder
    director.build_standard_report()
    marketing_report = marketing_builder.get_report()
    marketing_report.display()
    
    # Construindo relatório financeiro mínimo
    print("\n=== RELATÓRIO FINANCEIRO MÍNIMO ===")
    financial_builder = FinancialReportBuilder()
    director.builder = financial_builder
    director.build_minimal_report()
    financial_report = financial_builder.get_report()
    financial_report.display()

if __name__ == "__main__":
    main()

Vantagens

O Padrão Builder oferece várias vantagens significativas:

  • Construção Passo a Passo: Permite construir objetos complexos gradualmente
  • Reutilização de Código: O mesmo código de construção pode criar diferentes representações
  • Separação de Responsabilidades: Isola o código de construção da lógica de negócio
  • Controle sobre o Processo: Oferece mais controle sobre o processo de construção
  • Flexibilidade: Permite criar diferentes variações de um objeto
Anterior Próximo

Desvantagens

O Padrão Builder também apresenta algumas desvantagens:

  • Complexidade: Aumenta o número de classes no código
  • Overhead: Pode ser complexo demais para objetos simples
  • Manutenção: Requer manutenção de múltiplas classes
  • Acoplamento: Produtos e builders podem ficar acoplados
  • Performance: Processo de construção pode ser mais lento
Anterior Próximo

Conclusão

O Padrão Builder é uma ferramenta poderosa para construir objetos complexos passo a passo, especialmente quando esses objetos têm muitas partes configuráveis ou quando o processo de construção envolve múltiplas etapas.

Na nossa analogia da Pink, o Builder funciona como o processo de criação de performances - permitindo combinar diferentes elementos (vocais, visuais, musicais, cênicos) de maneira flexível para criar shows únicos e memoráveis.

Assim como a Pink constrói suas performances icônicas elemento por elemento, o padrão Builder nos permite construir objetos complexos de maneira controlada e organizada, garantindo que cada "performance" do nosso código seja perfeita.
