Padrões Criacionais

Padrões que abstraem o processo de instanciação de objetos, tornando o sistema independente de como os objetos são criados.

Design Patterns Orientação a Objetos Boas Práticas

Principais Padrões

1

Singleton

Garante que uma classe tenha apenas uma instância e fornece um ponto global de acesso a ela.

Ver Detalhes
2

Factory Method

Define uma interface para criar um objeto, mas permite que as subclasses decidam qual classe instanciar.

Ver Detalhes
3

Abstract Factory

Fornece uma interface para criar famílias de objetos relacionados sem especificar suas classes concretas.

Ver Detalhes
4

Builder

Separa a construção de um objeto complexo de sua representação, permitindo a mesma construção criar diferentes representações.

Ver Detalhes
5

Prototype

Especifica os tipos de objetos a serem criados usando uma instância prototípica e cria novos objetos copiando este protótipo.

Ver Detalhes
